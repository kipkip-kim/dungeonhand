## Lessons Learned

### What Went Well ✅

1. **설계 기반 구현**: 설계 문서가 명확해서 구현 편차 최소화 (최초 82% → 최종 98%)
2. **모듈화 설계**: data.js (상수) + utils.js (알고리즘) + screens/ (UI) 분리로 유지보수성 좋음
3. **반복적 개선**: Iteration 1/2로 P0→P1/P2 버그 체계적으로 해결 (82%→90%→98%)
4. **Stale closure 방지**: 이전 세션들의 패턴을 따라 useRef 방어적 추가
5. **battleNum 완전 제거**: grep으로 0건 검증, 완전한 구조 전환 달성

### Areas for Improvement 🔧

1. **초기 구현 품질**: 5건의 P0 버그가 발생 → 구현 전 체크리스트 강화 필요
   - 설계서 각 섹션별 구현 체크리스트 작성
   - 이벤트/callback 호출 검증 단계 추가
2. **노드 크기 상수화**: 42px vs 44px (cosmetic) → 설계/구현 조율 개선
3. **saveRun 상수 참조**: v: 2 (리터럴) → v: SAVE_VERSION으로 변경 (코드 품질)

### To Apply Next Time ✅

1. **구현 체크리스트 작성**: 설계의 각 섹션별로 구현/검증 항목 리스트 생성
2. **Event/Callback 검증**: 신규 이벤트 추가 시 "호출 지점" 검증 의무화
3. **상수 정의 규칙**: 매직 넘버 금지, 모든 수치는 상수로 정의
4. **초기 테스트**: 각 이벤트를 임플레이로 즉시 검증 (setDeck 같은 side effect 확인)

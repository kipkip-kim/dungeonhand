# Design: 지도 시스템 (Slay the Spire 스타일)

## Executive Summary

| 관점 | 내용 |
|------|------|
| **문제** | 층당 전투 2회, 일직선 고정 경로 → 단조로움 |
| **해결** | 7열 노드 맵 + 경로 선택 + 신규 몬스터/이벤트 |
| **기능/UX 효과** | 층당 전투 3~4회, 전략적 경로 선택, 리플레이 가치 |
| **핵심 가치** | 매 런마다 다른 경험 → 전략 깊이 + 수명 연장 |

---

## 1. 맵 구조 설계

### 1.1 7열 맵 레이아웃

```
열0(시작) → 열1 → 열2 → 열3 → 열4 → 열5 → 열6(보스)
             2~3   2~3   2~3   2~3   2~3     1
```

- **전 층 7열 균일** (열0 시작점 + 열1~5 선택 + 열6 보스)
- 열당 **2~3개 노드**, 보스열은 1개
- 플레이어는 경로당 **7개 노드** 방문

### 1.2 노드 타입

| 타입 | 아이콘 | 설명 | 열 배치 규칙 |
|------|--------|------|-------------|
| `battle` | ⚔️ | 일반 전투 | 어디든 (가장 흔함) |
| `elite` | 💪 | 미니보스 (강화 보상) | 열3~4에 1~2개 |
| `campfire` | 🔥 | 회복 + 이벤트 | 열2, 열5에 배치 |
| `shop` | 🏪 | 상점 | 열3~4에 1개 |
| `event` | ❓ | 랜덤 이벤트 | 열1~5 어디든 |
| `boss` | 👑 | 보스 (고정) | 열6 (항상 1개) |

### 1.3 노드 분배 규칙

```
열1: 3노드 — battle 2~3 + event 0~1
열2: 3노드 — battle 1~2 + campfire 1 + event 0~1
열3: 2노드 — elite 1 + (shop 또는 battle)
열4: 3노드 — battle 1~2 + shop 0~1 + event 0~1
열5: 2노드 — battle 1 + campfire 1
열6: 1노드 — boss (고정)
```

**경로당 예상 구성**: 전투 3~4회, 캠프 1~2회, 이벤트 0~1회, 상점 0~1회, 보스 1회

### 1.4 연결 규칙

- 각 노드는 다음 열의 **1~2개** 노드에 연결
- 모든 노드는 **최소 1개** 수신 연결 (데드엔드 방지)
- 열6(보스)는 열5의 **모든** 노드에서 연결
- 시작점(열0)은 열1의 **모든** 노드에 연결

### 2.3 몬스터 풀 확장

**현재**: 층당 4종 (일반2 + 미니보스1 + 보스1) = 20종
**변경**: 층당 6종 (일반4 + 미니보스1 + 보스1) = 30종

각 층에 **일반 몬스터 2종 추가**:

| 층 | 현재 일반 | 추가 일반 | 미니보스 | 보스 |
|----|----------|----------|---------|------|
| 1. 고블린 소굴 | 고블린, 궁수 | 고블린 도둑, 고블린 주술사 | 대장 | 킹 |
| 2. 언데드 묘지 | 해골, 뱀파이어 | 구울, 레이스 | 망령 기사 | 리치 |
| 3. 마법 탑 | 골렘, 마녀 | 가고일, 원소술사 | 불꽃 정령 | 대마법사 |
| 4. 심연 | 포식자, 심연눈 | 심연거미, 공허흡수자 | 사도 | 군주 |
| 5. 드래곤 둥지 | 알지기, 새끼 | 용암도마뱀, 드래곤 사제 | 근위병 | 로드 |

**MONSTERS 배열 구조 변경**:
- 현재: `(fl-1)*4 + slotIdx` → 인덱스 0~19
- 변경: `(fl-1)*6 + slotIdx` → 인덱스 0~29
- `MONSTERS_PER_FLOOR = 4` → `6`

## 3. 맵 생성 알고리즘 (utils.js)

```javascript
function generateFloorMap(floor) {
  // 1. 열 구조 생성 (7열)
  // 2. 각 열에 노드 배치
  //    - 열1: 3노드 (battle 위주)
  //    - 열2: 3노드 (campfire 1개 보장)
  //    - 열3: 2노드 (elite 1개 보장)
  //    - 열4: 3노드 (shop 가능)
  //    - 열5: 2노드 (campfire 1개 보장)
  //    - 열6: 1노드 (boss)
  // 3. 몬스터 인덱스 랜덤 배정 (일반 0~3 중 랜덤, elite=4, boss=5)
  // 4. 연결 생성
  //    - 각 노드 → 다음 열 1~2개 연결
  //    - 모든 노드 최소 1개 수신 보장
  //    - 열0→열1: 모든 연결
  //    - 열5→열6: 모든 연결
  // 5. return { rows, currentRow: 0, currentNodeIdx: 0 }
}
```

### 연결 생성 상세

```
노드 위치(열 내 인덱스)를 기준으로:
- 같은 인덱스의 다음 열 노드에 연결 (직진)
- 50% 확률로 인접 인덱스에도 연결 (분기)
- 마지막에 수신 연결 0인 노드 → 이전 열 가장 가까운 노드에서 연결 추가
```

## 4. 화면 흐름 변경

### 4.1 현재 흐름

```
startRun → battle(bn=1) → reward → battle(bn=2) → reward →
campfire → battle(bn=4) → reward → battle(bn=5) → relicReward → shop →
nextFloor(or victory)
```

### 4.2 변경 흐름

```
startRun → generateMap → MAP SCREEN → 노드 선택 →
  battle → reward → MAP SCREEN → 노드 선택 →
  campfire → MAP SCREEN → 노드 선택 →
  shop → MAP SCREEN → 노드 선택 →
  event → MAP SCREEN → 노드 선택 →
  boss → relicReward → nextFloor(or victory) → generateMap → MAP SCREEN ...
```

### 4.3 핵심 변경: advanceBattle → goToMap

```javascript
// 현재: advanceBattle(d) — battleNum++ 후 자동 분기
// 변경: goToMap() — 맵 화면으로 돌아감

function goToMap() {
  // 현재 노드 visited 표시
  // currentRow++ (다음 열로)
  // screen = "map"
  saveRun(); // 자동저장
}

// 보스 전투 승리 후:
function afterBossVictory() {
  // relicReward → shop 없이 바로 nextFloor or victory
  // (shop은 맵 노드로 이동)
}
```

### 4.4 screen 추가

현재 screens: menu, classSelect, village, battle, reward, enhance, relicReward, campfire, shop, victory, defeat
추가: **"map"**, **"event"** (또는 event를 오버레이로)

- 맵은 **가로 스크롤** (열이 좌→우로 진행)

- 노드 터치 영역 최소 44x44px

## 6. battleNum 제거 계획

### 6.1 현재 battleNum 사용처 (10곳+)

| 위치 | 용도 | 대체 |
|------|------|------|
| `beginBattle` 몬스터 선택 | `BATTLE_TO_SLOT[bn]` | `floorMap.currentNode.monIdx` |
| `beginBattle` BGM 분기 | `bn===5 → boss` | `node.type === "boss"` |
| `advanceBattle` 분기 | `bn<5, bn===3` | `goToMap()` (항상 맵으로) |
| `winBattle` 골드 계산 | `bn===5, bn===4` | `node.type` 기반 |
| `winBattle` 보스 포인트 | `BOSS_POINTS[monIdx]` | 유지 (monIdx 방식 변경) |
| `loadRun` 복원 | `bn===3/5` | `screen` state로 복원 |
| `saveRun` 저장 | `battleNum` 저장 | `floorMap` 저장 |
| `getBattleBg` | `battleNum===5` | `node.type === "boss"` |
| 상점 진입 | 보스 후 자동 | 맵 노드로 진입 |
| 캠프파이어 진입 | `bn===3` | 맵 노드로 진입 |

### 6.2 battleNum → nodeType 전환 전략

1. `battleNum` state 유지하되 의미 변경: 현재 노드의 전투 번호가 아닌, 층 내 전투 카운트
2. 또는 `battleNum` 완전 제거 → `floorMap.currentNode.type`으로 모든 분기

→ **완전 제거 권장** (깔끔한 설계)

## 7. 저장/불러오기 변경

### saveRun 확장

```javascript
function saveRun() {
  saveMeta();
  try {
    localStorage.setItem("dh_run", JSON.stringify({
      classId, floor, gold, hp,
      deck, relics, discardedRelicIds,
      bossesKilled, passiveState, poison, tenacityUsed,
      maxCardId,
      floorMap,        // 추가: 맵 전체
      currentScreen,   // 추가: 현재 화면 (map/battle/shop 등)
      v: SAVE_VERSION  // 버전 2로 올림
    }));
  } catch (e) {}
}
```

### loadRun 복원

```javascript
function loadRun() {
  // ... 기존 state 복원 ...
  if (data.floorMap) {
    setFloorMap(data.floorMap);
    // currentScreen 기반으로 화면 복원
    setScreen(data.currentScreen || "map");
  } else {
    // v1 세이브 호환: 기존 battleNum 기반 복원
  }
}
```

# gitt — Git 커밋 + 푸시 자동화

## 실행 순서

### Step 1: 스펙 스냅샷 갱신
WORK_LOG.md 파일 상단의 `## 현재 게임 스펙 스냅샷` 섹션을 현재 코드 기준으로 갱신한다.
코드에서 직접 읽어서 다음 항목을 표로 정리한다:

- 유물 목록 (이름, 효과, 티어)
- 몬스터 HP 테이블 (이름, HP)
- 메타 업그레이드 목록 (이름, 효과, cost, max)
- 카드 목록 (이름, 효과)
- 경제 수치 (전투별 골드 수입, 상점 가격표)
- 핵심 밸런스 수치 (플레이어 HP, 족보별 기본 배율 등)

스냅샷 맨 위에 갱신 일시를 명시한다.
이전 스냅샷은 덮어쓴다 (이력은 git이 관리).

### Step 2: git add
`git add .` 실행

### Step 3: 커밋 메시지 결정
- `$ARGUMENTS`가 있으면 → 그대로 커밋 메시지로 사용
- `$ARGUMENTS`가 비어있으면 → `git diff --cached --stat`을 읽고, 변경 내용을 한국어 한 줄로 요약해서 커밋 메시지 생성

### Step 4: git commit
`git commit -m "메시지"` 실행

### Step 5: git push
`git push` 실행

### Step 6: 완료 출력
커밋 해시, 메시지, 스냅샷 갱신 여부를 출력한다.

## 커밋 메시지 자동 생성 규칙

- 한국어로 작성
- 50자 이내
- "무엇을 했는지"를 명확하게
- 예: "유물 인벤토리 3칸 제한 구현"
- 예: "상점 리롤 시스템 추가"

## 스냅샷 형식 예시

```markdown
## 현재 게임 스펙 스냅샷
> 갱신: 2026-03-04

### 유물 (10종)
| 이름 | 효과 | 티어 |
|------|------|------|
| 낡은 숫돌 | 카드당 공격력+1 | 1 |
| ... | ... | ... |

### 몬스터 HP
| 이름 | HP |
|------|-----|
| 고블린 | 36 |
| ... | ... |

(이하 동일 형식)
```

## 사용 예시

```
/gitt 세션1 인벤토리 시스템 구현
→ 스냅샷 갱신 → git commit -m "세션1 인벤토리 시스템 구현" → git push

/gitt
→ 스냅샷 갱신 → (변경 내용 자동 요약) → git commit → git push
```
